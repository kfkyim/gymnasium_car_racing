# PPO for CarRacing-v3 (Gymnasium)

A personal fork of [xtma/pytorch_car_caring](https://github.com/xtma/pytorch_car_caring), modified and extended for learning purposes. This project trains a **Proximal Policy Optimization (PPO)** agent with a **Beta action distribution** to solve the continuous-control [CarRacing-v3](https://gymnasium.farama.org/environments/box2d/car_racing/) environment from Gymnasium (Box2D).

## Overview

There are two main model variants in this repo:

| Variant | Files | Description |
|---|---|---|
| **Frame-stacking CNN** | `train.py` / `test.py` | Original-style feedforward actor-critic that stacks 4 grayscale frames as input channels. |
| **LSTM** | `train_lstm.py` / `test_lstm.py` | Recurrent actor-critic that processes single frames and carries temporal context through an LSTM instead of frame stacking. |

Both retain the general structure of xtma's original code (especially the `Env()` wrappers), with the modifications described below.

### `train.py` / `test.py` changes vs. original

- **Gymnasium migration**: updated from the deprecated OpenAI `gym` to the maintained `gymnasium` library (uses `CarRacing-v3`, new 5-tuple `step()` API, and modern wrappers such as `GrayscaleObservation` / `NormalizeObservation`).
- **GAE (Generalized Advantage Estimation)**: added a `--lambda_` parameter for GAE-based advantage computation, plus explicit `die`/`done` handling so the GAE trace resets correctly at episode boundaries.
- **Quality of life**: checkpoint saving/loading (episode number, best score, running score, seed) for resuming training (`--load-weights`, `--params-path`), configurable `--max-episode-steps` and `--lr`, and more.

### `train_lstm.py` / `test_lstm.py` changes vs. original

- **LSTM instead of frame stacking**: single grayscale frames are fed through the CNN, then into an LSTM that maintains hidden/cell state across timesteps (with state zeroing at episode boundaries). Two CNN backbones are available via `--model net` (original deep CNN) and `--model net2` (Nature-DQN-style CNN).
- **PPO implementation details**: incorporates many of the recommendations from [The 37 Implementation Details of Proximal Policy Optimization](https://iclr-blog-track.github.io/2022/03/25/ppo-implementation-details/) supported by existing research, including orthogonal weight initialization with appropriate gains, layer-wise init for policy/value heads, learning-rate annealing (`--anneal-lr` with `--lr-floor`), gradient clipping (`--max-grad-norm`), target-KL early stopping of PPO epochs (`--target-kl`), configurable value-function coefficient (`--vf-coef`), reward scaling (`--scale-rewards`), and rollout-style buffer updates (`--unroll-steps`).
- **TensorBoard logging**: uses `torch.utils.tensorboard.SummaryWriter` (runs saved under `runs/`) instead of visdom for tracking scores and losses.

## Installation

```bash
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt
```

> Note: PyTorch is installed from the CUDA wheel index; adjust `requirements.txt` for CPU-only installs or different supported CUDA versions.

## Usage

### Training (frame-stacking model)

```bash
python train.py --gamma 0.99 --lambda_ 0.95 --lr 1e-4 --action-repeat 6 --img-stack 3
```

Resume from a checkpoint:

```bash
python train.py --load-weights --params-path checkpoints/...
```

### Training (LSTM model)

```bash
python train_lstm.py --model-name my_run --model net2 --lr 2.5e-4 --anneal-lr --vf-coef 0.5 --action-repeat 4 --unroll-steps 128 --ppo-epochs 5
```

Monitor with TensorBoard:

```bash
tensorboard --logdir runs
```

### Testing

```bash
python test.py --params-path checkpoints/... --num-episodes 5 --render
python test_lstm.py --params-path checkpoints/my_run.pkl --model net2 --num-episodes 3
```

Checkpoints are saved to `checkpoints/`; TensorBoard logs to `runs/`.
